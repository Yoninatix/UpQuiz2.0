# UpQuiz — Deployment & Setup Guide

> **Thesis project** | CCS Research 2026  
> Stack: Debian Trixie · PostgreSQL 16 · Milvus 2.4 · Nginx · Ollama + Gemma 3 4B · RAG Pipeline

## Architecture Overview

```
Browser
  │
  ▼
Nginx (port 80/443)  ← reverse proxy, Debian Trixie
  ├─► /api/*    → Go Backend (Gin, port 8080)
  ├─► /ai/*     → AI / RAG Service (Node.js, port 3001)
  └─► /*        → React SPA (static, port 80)

Go Backend ──► PostgreSQL 16  (relational data: users, exams, attempts)
AI Service ──► Milvus 2.4     (vector DB: document embeddings for RAG)
AI Service ──► Ollama          (LLM inference: Gemma 3 4B)
               └─► etcd + MinIO (Milvus dependencies)
```

### Security Model (HMAC)
Tokens are signed with **HMAC-SHA256 (HS256)** via `golang-jwt/jwt`. The JWT secret (`JWT_SECRET`) must be a cryptographically random string (≥ 64 hex characters). Tokens are delivered as **HTTP-only cookies** — no `localStorage` is used, which eliminates XSS-based token theft.

## Quick Start (Development)

### 1. Clone and configure

```bash
git clone <your-repo-url>
cd UpQuiz-Thesis-

# Copy the example env file and fill in secrets
cp .env.example .env
```

### 2. Generate a secure HMAC secret

```bash
# Linux / macOS / WSL
openssl rand -hex 64
```

Copy the output into `.env` as the value for `JWT_SECRET`.

### 3. Edit `.env`

```env
# ── PostgreSQL ─────────────────────────────────
POSTGRES_DB=examdb
POSTGRES_USER=examuser
POSTGRES_PASSWORD=your_strong_password_here

# ── HMAC / JWT ─────────────────────────────────
JWT_SECRET=<paste openssl output here>
JWT_EXPIRY_HOURS=24
COOKIE_SECURE=false       # true when running behind HTTPS
COOKIE_SAME_SITE=Lax

# ── Ollama / LLM ───────────────────────────────
OLLAMA_MODEL=gemma3:4b    # ~2.5 GB download on first run

# ── Milvus ─────────────────────────────────────
MILVUS_COLLECTION=document_chunks
```

### 4. Build and start all services

```bash
docker compose up --build -d
```

The first run will:
- Build Go, Node.js, and Nginx images from source
- Download the Ollama runtime
- Pull the **Gemma 3 4B** model (~2.5 GB) — this can take 5–15 minutes depending on your connection
- Initialize PostgreSQL with database migrations

### 5. Verify everything is running

```bash
docker compose ps
```

Expected: all services `healthy` or `running`.

```bash
# Quick smoke test
curl http://localhost/nginx-health         # → "healthy"
curl http://localhost/api/health           # → {"status":"ok"}
```

Open your browser at **http://localhost**.

## Service Details

### PostgreSQL 16
- Container: `examdb-postgres`
- Internal host: `postgres:5432`
- Migrations auto-applied from `./database/migrations/` on first start
- Data persisted in Docker volume `postgres_data`

### Milvus 2.4 (Vector Database)
- Container: `milvus-standalone`
- gRPC port (internal): `19530`
- Depends on: **etcd** (metadata) + **MinIO** (object storage)
- Collection: configured by `MILVUS_COLLECTION` (default: `document_chunks`)
- Stores 768-dimensional embeddings generated by `nomic-embed-text`

### Ollama + Gemma 3 4B (LLM Inference)
- Container: `ollama`
- REST API (internal): `http://ollama:11434`
- Default model: **gemma3:4b** (~2.5 GB on disk)
- Startup container `ollama-setup` automatically pulls the model on first run

#### Switching models

Edit `OLLAMA_MODEL` in `.env` before running `docker compose up`:

| Model | Size | Notes |
|---|---|---|
| `gemma3:4b` | ~2.5 GB | **Recommended** — best quality/size ratio |
| `gemma3:1b` | ~800 MB | Lighter, lower RAM systems |
| `llama3.2:3b` | ~2 GB | Meta's 3B model, strong reasoning |
| `phi3:mini` | ~2.3 GB | Microsoft Phi-3, good for instruction tasks |

### Nginx (Reverse Proxy)
- Container: `exam-nginx`
- Ports: `80` (HTTP), `443` (HTTPS — configure TLS cert manually)
- Routes: `/api/*` → backend, `/ai/*` → AI service, `/*` → React SPA
- Upload limit: 50 MB (configurable in `nginx/nginx.conf`)
- Built from **Debian Trixie slim**

### Go Backend
- Container: `exam-backend`
- Built from `golang:1.23-bookworm` (build) → `debian:trixie-slim` (runtime)
- Auth: HMAC-SHA256 JWT via `golang-jwt/jwt/v5`
- Tokens carried as HTTP-only cookies (no `localStorage`)

### AI / RAG Service
- Container: `exam-ai-service`
- Built from `node:20-bookworm` (build) → `debian:trixie-slim` (runtime)
- Accepts PDF/DOCX uploads → chunks → embeds with `nomic-embed-text` → stores in Milvus
- On quiz generation: retrieves top-k chunks → feeds as context to Gemma → returns questions

## RAG Pipeline (How It Works)

```
Upload PDF/DOCX
       │
       ▼
  Text Extraction (pdf-parse / mammoth)
       │
       ▼
  Chunking (sliding window, ~500 tokens)
       │
       ▼
  Embedding (nomic-embed-text via Ollama → 768-dim vectors)
       │
       ▼
  Store in Milvus collection
       │
─ ─ ─ ─ ─ ─ ─ ─ ─
Query (generate quiz)
       │
       ▼
  Embed query → similarity search in Milvus
       │
       ▼
  Top-k chunks retrieved as context
       │
       ▼
  Prompt Gemma 3 4B with context (RAG)
       │
       ▼
  Return generated questions via API
```

## HMAC Authentication Flow

```
1. POST /api/auth/login  { email, password }
         │
         ▼
   Backend verifies password hash (bcrypt)
         │
         ▼
   Signs JWT with HMAC-SHA256 (HS256) using JWT_SECRET
         │
         ▼
   Sets HTTP-only cookie: token=<jwt>; HttpOnly; SameSite=Lax
         │
         ▼
   Browser stores cookie automatically — NO localStorage

2. Subsequent requests:
   Browser sends cookie automatically
         │
         ▼
   Backend middleware reads cookie, validates HMAC signature
         │
         ▼
   Injects userID + role into request context
```

## Production Checklist

- [ ] Set `JWT_SECRET` to a 64-byte random hex string (`openssl rand -hex 64`)
- [ ] Set strong `POSTGRES_PASSWORD`
- [ ] Set `COOKIE_SECURE=true` (requires HTTPS)
- [ ] Configure TLS certificate in Nginx (`./nginx/conf.d/default.conf`)
- [ ] Change MinIO `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` from defaults
- [ ] Set `GO_ENV=production` in `.env`
- [ ] Restrict Docker network exposure — only Nginx ports should be public
- [ ] Consider GPU passthrough for Ollama (see commented block in `docker-compose.yml`)

### HTTPS with Nginx

Add your certificate files and update `nginx/conf.d/default.conf`:

```nginx
server {
    listen 443 ssl;
    ssl_certificate     /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    # ... rest of your config
}
server {
    listen 80;
    return 301 https://$host$request_uri;
}
```

Mount the certs in `docker-compose.yml`:

```yaml
nginx:
  volumes:
    - ./nginx/conf.d:/etc/nginx/conf.d:ro
    - ./nginx/ssl:/etc/nginx/ssl:ro
```

## Troubleshooting

| Problem | Likely cause | Fix |
|---|---|---|
| `milvus-standalone` exits immediately | etcd/MinIO not ready | Run `docker compose up` again; healthchecks retry automatically |
| Ollama model pull hangs | Slow internet | Be patient — Gemma 3 4B is ~2.5 GB. Check `docker compose logs ollama-setup` |
| Backend can't connect to postgres | DB not yet initialized | Wait for `examdb-postgres` to show `healthy` in `docker compose ps` |
| 413 Request Entity Too Large | File > 50 MB | Increase `client_max_body_size` in `nginx/nginx.conf` |
| JWT cookie not sent | `COOKIE_SECURE=true` on HTTP | Use HTTPS or set `COOKIE_SECURE=false` for dev |
| Vector search returns no results | Collection empty | Upload a document first via the UI |
